## 参考答案（示例）

1. Prompt 决定 LLM 的行为边界和输出风格，直接影响 Agent 执行质量。
2. 角色定义、能力边界、行为准则、输出格式、示例（可选）。
3. 解决输出风格和格式不稳定问题。
4. 复杂推理或多步计算任务。
5. ReAct 会在推理过程中结合工具调用和观察结果迭代决策。
6. 因为下游程序要消费输出，结构化数据更易解析和自动化处理。
7. 增加格式约束、增加 few-shot、降低 temperature、增加解析兜底。
8. 用统一测试集做对比，观察正确率、格式稳定性、越界率等指标。
9. 角色定义、能力边界、工具调用规则、输出格式、安全护栏、错误恢复、上下文管理等。
10. 压缩重复规则、减少冗长示例、只保留代表性 few-shot、摘要历史上下文、用检索替代全量塞入。
11. 不是。示例过多会增加成本，也可能引入冲突；应选择覆盖关键边界的代表性样本。
12. 简单任务、低延迟任务、需要简洁输出或不适合暴露详细推理过程的场景。
13. 多样性、一致性、渐进性、数量适中，覆盖正例、反例和边界情况。
14. “请一步一步思考”或 “Let's think step by step”。
