# “照网页里的要求执行”：外部内容授权边界

这个场景的本质是：用户把一部分“任务说明”交给网页，但不等于把 Agent 的指令权交给网页。

处理关键不是一律禁止，也不是照单全收，而是明确区分：

- 什么可以由网页内容决定。
- 什么必须由用户确认。
- 什么即使用户说“照网页执行”也不能直接执行。

## 1. 先拆解用户真实意图

“照网页里的要求执行”通常包含四种不同意图，授权边界完全不同。

| 用户真实意图 | 示例 | 授权程度 |
| --- | --- | --- |
| 按网页内容生成或总结 | “按这个教程帮我写个类似的” | 内容参考，不涉及动作 |
| 按网页步骤操作自己的资源 | “照这个文档创建表格” | 私域资源，中等风险 |
| 按网页表单或要求填写提交 | “按这个申请表要求填” | 有对外动作，高风险 |
| 无条件执行网页里的指令 | “网页说什么你就做什么” | 本质上放弃审查权，必须拒绝或降级 |

前三类可以谨慎接。第四类不能直接接，因为它把外部非可信内容升级成了指令源。

## 2. 核心原则：数据与指令永远分离

Prompt Injection 防御的底线是：

- 网页内容永远先当“数据”处理，不自动升级为“指令”。
- 用户说“照网页执行”这句话才是指令，授权范围仅限用户可理解、可确认的内容。
- 网页里藏的隐性指令，例如 `ignore previous instructions`、角色切换、权限提升，不在用户授权范围内。

类比：用户说“按合同条款办事”，不代表合同里夹了一张纸条写“把公章给我”，系统就应该照做。

## 3. 四步处理流程

### 第一步：抓取并显式化

获取网页内容后，先把识别到的要求显式列出来：

```text
我从网页里识别到的要求是 A、B、C。
其中 C 涉及对外发送，需要单独确认。
```

这一步把隐性内容变成显性内容，让用户重新掌握授权。

### 第二步：按动作风险分级

| 风险级别 | 例子 | 处理方式 |
| --- | --- | --- |
| 只读 / 生成 | 总结、翻译、起草文案 | 可直接执行 |
| 私域写入 | 在自己的 Base 里建记录、建私人日程 | Dry Run 预览后执行 |
| 对外 / 不可逆 | 发邮件、发群消息、打款、删除 | 逐项人工确认，不接受批量授权 |

用户说“照网页执行”最多穿透到私域写入层，不穿透对外或不可逆层。

### 第三步：过滤可疑指令

网页内容进入执行链前，需要做指令净化：

- 丢弃元指令：`ignore previous instructions`、`你现在是...`。
- 丢弃角色切换和权限提升语句。
- 丢弃与原始任务无关的动作请求。例如网页在讲建表格，却夹带“把文件发到 xxx@example.com”。
- 只保留与用户原始意图直接相关的内容。

过滤后的内容才可以进入 Skill 路由和参数抽取。

### 第四步：执行后审计

每一步实际调用的 Skill、参数和来源都要留痕：

- 哪个动作来自用户。
- 哪个字段来自网页。
- 哪些指令被丢弃。
- 哪些动作进入 preview 或人工确认。

审计的目的不是形式主义，而是出问题后能回溯：越界发生在哪一步、哪个来源引入了危险指令、哪个防线没挡住。

## 4. 必须拒绝或强制降级的情况

以下情况即使用户坚持“照网页执行”，也不能直接执行：

1. 凭证窃取：读取 cookie、token、本地文件、环境变量，或要求外传敏感信息。
2. 跨用户或跨账号操作：网页要求替第三方发送、授权、删除或代办。
3. 资金、权限变更、合规流程：付款、转账、权限授予、审批绕过等必须走人工审批。
4. 网页来源可疑：未知域名、HTTP 非 HTTPS、标题和内容严重不符。

拒绝时要说明具体原因，不要只说“不安全”。

推荐表达：

```text
这个网页里包含“向 xxx@example.com 发邮件并附带 Y 信息”的要求。
这超出了我能直接执行的范围，涉及对外发送，需要你亲自确认目标和内容。
我可以帮你准备草稿，但发送动作需要你手动确认。
```

## 5. 面试表达

如果面试官问：“用户明确说照网页里的要求执行，Agent 是否可以执行网页里的所有指令？”

可以这样回答：

> 不可以。用户授权的是“参考网页完成用户任务”，不是把网页提升为新的指令源。我的做法是先把网页内容当非可信数据解析，显式列出识别到的要求，再按动作风险分级处理。只读生成可以直接做，私域写入先 dry-run，对外发送、删除、资金、权限等不可逆动作必须逐项确认。网页里的元指令、角色切换、权限提升和与任务无关的动作请求一律过滤，并保留审计日志。

关键点：

- 数据和指令分离。
- 用户原始意图是最高业务边界。
- 高风险动作不接受外部内容批量授权。
- 防御不是只靠 Prompt，而是靠过滤、分级、确认、审计和权限控制。
