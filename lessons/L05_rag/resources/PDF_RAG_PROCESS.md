# PDF RAG 切分处理流程

PDF 做 RAG 的核心原则不是“转成纯文本后直接切”，而是先做版面解析和结构还原，再决定哪些信息进入 chunk、哪些进入 metadata、哪些作为独立资源索引、哪些可以丢弃。

## 1. PDF 信息分类

| 信息类型 | 推荐处理 |
| --- | --- |
| 正文文本 | 清洗后进入主 chunk |
| 标题/章节/页码/文档名 | 进入 metadata，必要时拼入 chunk 前缀 |
| 图片/图表/表格 | 转成可检索文本描述或结构化数据 |
| 页眉/页脚/水印 | 无语义重复内容删除，有价值信息入 metadata |
| 脚注/图注/表注 | 与对应正文、图表或表格关联 |

## 2. 页眉页脚

常见页眉页脚包括公司名、文档标题、章节名、页码、版权、保密等级、日期和文件路径。

风险：

- 重复页眉页脚会污染 embedding。
- 大量无效重复文本会浪费上下文。

推荐处理：

- 页码、版权声明、固定水印、文件路径一般删除。
- 文档标题、章节名、版本号、适用地区、密级、生效日期放入 metadata。
- 检索强依赖版本和章节时，可以把少量 metadata 拼入 chunk 前缀。

示例 metadata：

```json
{
  "doc_title": "支付风控系统接入指南",
  "section": "3.2 交易风控策略",
  "page": 12,
  "version": "v2.1",
  "effective_date": "2026-05-01",
  "security_level": "internal"
}
```

## 3. 图片

图片需要先分类：

- 装饰图：logo、背景、分隔图，通常删除。
- 扫描文字：OCR 后按正文处理。
- 产品截图：OCR + UI 结构描述。
- 流程图/架构图：抽取节点、边、方向、依赖关系。
- 业务图示：抽取流程、角色、审批路径。

图片 chunk 需要保留：

```json
{
  "modality": "image",
  "page": 8,
  "image_id": "img_008_02",
  "bbox": [120, 240, 520, 580],
  "caption": "图 3-2 支付风控请求链路"
}
```

## 4. 表格与可视化图表

表格不要直接按行抽文本，否则会丢表头和列关系。

推荐：

- 小表格整体作为一个 chunk。
- 大表格按行组切分，每个 chunk 必须带表头。
- 错误码、参数名、API 名、版本号等关键字段建立关键词或结构化索引。
- 数值分析类问题走 SQL/DataFrame/计算工具，不要只靠文本 RAG。

表格 chunk 示例：

```json
{
  "chunk_id": "doc1_p8_t1",
  "type": "table",
  "content": "表格：错误码说明\n列：错误码｜含义｜处理方式\n行：1001｜参数缺失｜检查必填字段",
  "metadata": {
    "page": 8,
    "caption": "表 2-1 错误码说明"
  }
}
```

可视化图表应抽取标题、图例、坐标轴、单位、数据点、趋势摘要和来源位置。

## 5. 图文关系

PDF 切分的关键不只是 OCR，而是图文关系是否保留。

建议：

- 根据版面分析恢复阅读顺序。
- 将图片/图表挂到最近标题或段落下。
- 保留图注/表注。
- 对“如下图所示”“见图 3-2”这类段落，与图片描述建立 `related_chunk_ids`。

## 6. 推荐流水线

```text
PDF 输入
  ↓
文档类型判断：原生 / 扫描 / 混合
  ↓
版面解析：标题、段落、表格、图片、页眉页脚、脚注
  ↓
阅读顺序重建
  ↓
页眉页脚 / 水印清洗
  ↓
OCR / 图像理解 / 表格解析
  ↓
生成结构化 block：text / table / image / chart
  ↓
语义切分
  ↓
metadata 注入
  ↓
多路索引：向量、关键词、表格结构化、图片描述
  ↓
检索 + rerank
  ↓
带来源的答案生成
```

## 7. 常见坑

- 直接按页切。
- 直接转纯文本，丢掉表格结构和图文关系。
- 页眉页脚不清洗，污染向量检索。
- 图片直接丢弃。
- 表格不带表头切分。
- 不保存页码和 bbox，无法引用和审计。
- 图表只写“有一张图”，没有抽取节点、关系、趋势和数据点。
- OCR 不标置信度。
- 忽略权限和敏感水印。

## 8. 面试回答模板

PDF 切分不能简单按页或固定 token 切。我会先做版面解析，识别标题、正文、表格、图片、页眉页脚和阅读顺序。重复的页码、版权、水印会清洗掉；文档标题、章节名、版本号等有价值信息会进入 metadata。图片按类型处理：装饰图删除，扫描文字 OCR，截图做 OCR 加视觉描述，架构图和流程图抽取节点、边和关系。表格保留表头和单元格关系，小表整体切，大表按行组切但每个 chunk 带表头。最后每个 chunk 保留页码、bbox、caption、section、related_chunk_ids，保证检索、引用和溯源可用。
