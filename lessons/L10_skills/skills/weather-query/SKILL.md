# Weather Query Skill

## 何时使用

- 用户问某个城市的天气。
- 用户问气温、是否下雨、是否需要带伞。
- 用户问空气质量、是否适合户外运动。
- 用户问出门穿什么衣服。

## 何时不使用

- 用户问航班或机票信息。
- 用户问旅游攻略。
- 用户只是闲聊天气但没有指定城市。

## 执行步骤

### Step 1: 解析城市

从用户消息中提取目标城市。若未指定城市，先追问用户。

### Step 2: 并行查询

同时调用：

- `get_weather(city)`：获取温度、湿度、天气状况、降雨概率。
- `get_air_quality(city)`：获取 AQI、PM2.5 和主要污染物。

### Step 3: 综合建议

| 条件 | 建议 |
| --- | --- |
| 气温 < 10°C | 注意保暖，建议穿厚外套或羽绒服 |
| 降雨概率 > 60% | 建议带伞 |
| AQI > 150 | 减少户外活动，建议戴口罩 |
| AQI < 50 且晴 | 适合户外运动 |

### Step 4: 输出格式

输出结构化天气报告，包含城市、温度、体感温度、湿度、天气、降雨概率、AQI、PM2.5 和出行建议。

## 注意事项

- 城市名优先使用中文，API 调用失败时尝试英文。
- 无法获取数据时明确告知用户，不要编造。
- 出行建议要具体，避免“注意天气”这类空泛表述。
