# 成本监控与安全防护

部署后的 Agent 不只是“能访问”，还要可控。

## 1. 成本指标

至少记录：

- 请求数。
- 输入 token。
- 输出 token。
- 平均延迟。
- P95/P99 延迟。
- 工具调用次数。
- 单请求估算成本。

成本异常常见原因：

- 简单问题走复杂 Agent。
- 工具调用死循环。
- 历史上下文无限增长。
- RAG 检索塞入过多 chunk。
- 没有缓存高频问题。

## 2. 成本控制方法

- 简单问题走轻量模型。
- 设置最大上下文长度。
- 限制最大工具步数。
- 对高频问题做缓存。
- RAG top_k 和 chunk 长度可配置。
- 长任务异步化，不占用同步请求。

## 3. 安全防护

输入侧：

- 检测 Prompt Injection 模式。
- 限制输入长度。
- 外部网页内容当数据，不当指令。
- 对敏感词和越权意图做识别。

执行侧：

- 高风险工具需要确认。
- 权限过滤前置到检索和工具层。
- 写操作使用幂等键。
- 审计日志记录每次工具调用。

输出侧：

- 不泄露 system prompt。
- 不输出密钥、token、cookie。
- 无证据时拒答。
- 对外发送前展示预览。

## 4. 监控闭环

推荐日志字段：

```json
{
  "session_id": "user_001",
  "request_id": "req_001",
  "latency_ms": 1200,
  "input_tokens": 500,
  "output_tokens": 200,
  "tool_calls": ["search_knowledge_base"],
  "safety_flags": [],
  "status": "ok"
}
```

有了结构化日志，才能做告警、成本看板、失败追踪和回归评估。
