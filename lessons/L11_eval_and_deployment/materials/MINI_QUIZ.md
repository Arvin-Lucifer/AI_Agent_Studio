# L11 小测

1. 为什么 Agent 不能只靠手工问几个问题就上线？
2. Agent 评估至少包含哪六个维度？
3. 为什么 `expected_tool` 要参与评测？
4. 关键词匹配会产生哪两类问题？
5. 安全评估为什么不应该只作为总分里的普通权重？
6. 幻觉、死循环、工具误调分别怎么修复？
7. `/health` 接口为什么不应该调用模型？
8. SSE 流式输出相比一次性返回有什么好处？
9. Nginx 代理 SSE 时为什么要关闭 `proxy_buffering`？
10. 部署后需要记录哪些成本和安全字段？

## 参考答案

1. 手工样例无法覆盖边界、安全、工具误调和长尾问题。
2. 准确性、完整性、相关性、安全性、效率/成本、鲁棒性。
3. Agent 的过程也重要，工具调错可能导致成本、安全和答案风险。
4. 假阳性和假阴性。
5. 严重安全失败应作为 hard gate，而不是被平均分掩盖。
6. 幻觉靠证据和拒答；死循环靠最大步数和重复检测；工具误调靠工具描述、路由和评测集。
7. 健康检查必须轻量可靠，不能依赖模型网关或产生费用。
8. 用户能更早看到进度，工具调用可见，长回答体验更好。
9. 否则代理可能缓存分块，导致浏览器不能实时收到 token。
10. session_id、latency、tokens、tool_calls、safety_flags、status、error_type。
